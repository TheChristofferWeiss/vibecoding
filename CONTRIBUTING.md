# Contributing

Thank you for your interest in contributing to this starter template! This document provides guidelines and instructions for contributing.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists in the [Issues](https://github.com/yourusername/nextjs-supabase-vercel-starter/issues) section
2. If not, create a new issue with:
   - Clear title and description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Environment details (OS, Node version, etc.)

### Suggesting Improvements

We welcome suggestions for:
- New features
- Documentation improvements
- Code optimizations
- Better examples

Open an issue with the `enhancement` label and describe your idea.

### Submitting Pull Requests

1. **Fork the repository**
2. **Create a feature branch:**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes:**
   - Follow the existing code style
   - Add comments for complex logic
   - Update documentation if needed
4. **Test your changes:**
   - Test locally with `npm run dev`
   - Ensure TypeScript compiles: `npm run type-check`
   - Check linting: `npm run lint`
5. **Commit your changes:**
   ```bash
   git commit -m "feat: Add your feature description"
   ```
   Use conventional commit messages:
   - `feat:` for new features
   - `fix:` for bug fixes
   - `docs:` for documentation
   - `refactor:` for code refactoring
   - `test:` for tests
6. **Push to your fork:**
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Open a Pull Request:**
   - Provide a clear description
   - Reference any related issues
   - Wait for review

## Code Style

- Use TypeScript for all new code
- Follow Next.js conventions
- Use functional components with hooks
- Keep components small and focused
- Add JSDoc comments for complex functions

## Documentation

When adding new features:
- Update the README if needed
- Add examples to relevant docs
- Update the troubleshooting guide if you fix common issues

## Questions?

Feel free to open an issue with the `question` label if you need help or clarification.
